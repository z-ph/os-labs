三、准备 cc switch 源文件

本课设使用开源项目 SaladDay/cc-switch-cli 的 Linux Release 包作为真实打包对象。你的 WSL Ubuntu 截图中 uname -m 输出为 x86_64，因此选择 linux-x64 包；如果在 ARM64/aarch64 的 Kylin 或 Linux 环境中操作，才选择 linux-arm64 包。一般不要选 musl 包，除非系统是 Alpine 这类 musl libc 环境。

cd ~/os-course/cc-switch-packaging
mkdir -p source
cd source

# x86_64 / amd64 环境选择 linux-x64
curl -LO https://github.com/SaladDay/cc-switch-cli/releases/download/v5.8.7/cc-switch-cli-v5.8.7-linux-x64.tar.gz
tar -zxvf cc-switch-cli-v5.8.7-linux-x64.tar.gz

# 如果 uname -m 显示 aarch64，再改用下面的 arm64 包：
# curl -LO https://github.com/SaladDay/cc-switch-cli/releases/download/v5.8.7/cc-switch-cli-v5.8.7-linux-arm64.tar.gz
# tar -zxvf cc-switch-cli-v5.8.7-linux-arm64.tar.gz

find . -maxdepth 3 -type f -perm -111 -print
if [ ! -f cc-switch ]; then
    cp "$(find . -maxdepth 3 -type f -perm -111 | head -n 1)" cc-switch
fi
chmod +x cc-switch
./cc-switch --version
./cc-switch --help
cd ..

命令与参数解释：

  • cc-switch-cli-v5.8.7-linux-x64.tar.gz 适用于 x86_64 / amd64 Linux 环境，例如你的 WSL Ubuntu。
  • cc-switch-cli-v5.8.7-linux-arm64.tar.gz 适用于 aarch64 / ARM64 环境。
  • find . -maxdepth 3 -type f -perm -111 查找解压目录中的可执行文件。
  • cp ... cc-switch 将真实 Release 可执行文件统一命名为 source/cc-switch，便于后续 DEB/RPM 打包流程复用。
  • --version 和 --help 用于证明打包对象是真实可执行的开源 CLI 程序。
截图内容：截取下载链接、解压结果、find 找到的可执行文件，以及 cc-switch --version / cc-switch --help 的输出，证明打包对象来自真实开源 Release 且可运行。
图片占位符：cc switch 开源 Release 验证截图
需要包含 linux-x64 下载包、解压后的可执行文件、--version 和 --help 输出
报告结论：本项目已从 cc-switch-cli 开源 Release 获取真实 Linux x64 可执行文件，并完成打包前运行验证，具备进入 DEB/RPM 适配打包流程的基础条件。

四、9.5.1 基于 DEB 格式的适配打包

阶段 1：创建 DEB 包目录结构

cd ~/os-course/cc-switch-packaging
rm -rf deb-root
mkdir -p deb-root/DEBIAN
mkdir -p deb-root/usr/local/bin
mkdir -p deb-root/usr/share/doc/cc-switch
install -m 0755 source/cc-switch deb-root/usr/local/bin/cc-switch
cat > deb-root/usr/share/doc/cc-switch/README.txt <<'EOF'
cc-switch is a course-design CLI packaged for Kylin OS adaptation testing.
EOF
find deb-root -maxdepth 4 -type f -o -type d

命令与参数解释：

  • DEBIAN 保存 DEB 包控制文件。
  • usr/local/bin 保存安装后的命令文件，安装后可直接运行 cc-switch。
  • install -m 0755 复制文件并设置可执行权限。
  • find 展示打包目录结构，是报告中非常重要的截图证据。
截图内容：截取 find deb-root 的目录结构，证明 DEB 包按规范组织了控制文件和程序文件。
图片占位符：DEB 目录结构截图
需要看到 DEBIAN、usr/local/bin/cc-switch、usr/share/doc/cc-switch
报告结论：DEB 打包目录已按 Linux 文件系统层次和 Debian 包规范完成组织。

阶段 2：编写 DEB 控制文件

cat > deb-root/DEBIAN/control <<'EOF'
Package: cc-switch
Version: 1.0.0
Section: utils
Priority: optional
Architecture: amd64
Maintainer: zph <zph@example.com>
Depends: bash
Description: cc switch command-line tool for Kylin adaptation packaging
 This package installs the cc-switch command for operating system course design.
 It is used to verify DEB packaging, installation, command execution and removal.
EOF
cat deb-root/DEBIAN/control

命令与参数解释：

  • Package 是软件包名称，安装和卸载时使用。
  • Version 是版本号，报告中可用于说明适配版本。
  • Architecture: amd64 对应 x86_64 / amd64 Linux 二进制；如果使用 ARM64 包，应改为 arm64。
  • Depends: bash 表示运行依赖。
截图内容：截取 control 文件内容，证明 DEB 元数据完整。
图片占位符：DEB control 文件截图
需要包含 Package、Version、Architecture、Depends、Description
报告结论：DEB 控制文件已定义软件包名称、版本、架构、依赖和说明信息，满足包管理器识别要求。

阶段 3：构建、安装与验证 DEB 包

dpkg-deb --build deb-root cc-switch_1.0.0_amd64.deb
ls -lh cc-switch_1.0.0_amd64.deb
sudo dpkg -i cc-switch_1.0.0_amd64.deb
dpkg -l | grep cc-switch
which cc-switch

命令与参数解释：

  • dpkg-deb --build 将目录构建为 .deb 文件。
  • sudo dpkg -i 安装 DEB 包。
  • dpkg -l | grep cc-switch 查询包是否已安装。
  • which cc-switch 确认命令已进入系统 PATH。

需要包含 .deb 文件、dpkg -i、dpkg -l、cc-switch --version 输出

五、9.5.2 基于 RPM 格式的适配打包

阶段 1：安装工具并初始化 rpmbuild 目录

sudo yum install -y rpm-build rpmdevtools || sudo dnf install -y rpm-build rpmdevtools
rpmdev-setuptree || mkdir -p ~/rpmbuild/{BUILD,BUILDROOT,RPMS,SOURCES,SPECS,SRPMS}
find ~/rpmbuild -maxdepth 1 -type d | sort

命令与参数解释：

  • rpm-build 提供 rpmbuild 命令。
  • rpmdevtools 提供 rpmdev-setuptree，用于自动创建打包目录。
  • BUILD、BUILDROOT、RPMS、SOURCES、SPECS、SRPMS 是 RPM 打包标准目录。

cd ~/os-course/cc-switch-packaging

Course-design CLI for Kylin RPM packaging adaptation.

ls -lh ~/rpmbuild/SOURCES/cc-switch-1.0.0.tar.gz

命令与参数解释：

  • tar -czf 创建 gzip 压缩源码包。
  • SOURCES 是 RPM 构建读取源码包的位置。

阶段 3：编写 RPM spec 文件

cat > ~/rpmbuild/SPECS/cc-switch.spec <<'EOF'
Name:           cc-switch
Version:        1.0.0
Release:        1%{?dist}
Summary:        cc switch command-line tool for Kylin adaptation packaging
License:        MIT
Source0:        %{name}-%{version}.tar.gz
Requires:       bash

%description
cc-switch is a course-design command-line tool packaged to verify RPM
adaptation, installation, command execution and removal on Kylin OS.

%prep
%setup -q

%build

%install
rm -rf %{buildroot}
mkdir -p %{buildroot}/usr/local/bin
mkdir -p %{buildroot}/usr/share/doc/%{name}
install -m 0755 cc-switch %{buildroot}/usr/local/bin/cc-switch
install -m 0644 README.md %{buildroot}/usr/share/doc/%{name}/README.md

%files
/usr/local/bin/cc-switch
/usr/share/doc/%{name}/README.md

%changelog
* Sat Jul 04 2026 zph <zph@example.com> - 1.0.0-1
- Initial RPM package for operating system course design
EOF
cat ~/rpmbuild/SPECS/cc-switch.spec

命令与参数解释：

  • Name、Version、Release 定义 RPM 包身份。
  • 本 spec 不设置 BuildArch: noarch，因为这里打包的是 Linux x64 二进制文件，RPM 应按实际构建平台生成 x86_64 包。
  • %prep 解压源码，%install 将文件安装到构建根目录，%files 声明最终打入包内的文件。
截图内容：截取 spec 文件，重点包含元数据、install 阶段和 files 清单。
图片占位符：RPM spec 文件截图

报告结论：RPM spec 文件已完整描述 cc switch 的源码、安装路径、运行依赖和文件归属。

阶段 4：构建、安装与验证 RPM 包

rpmbuild -ba ~/rpmbuild/SPECS/cc-switch.spec
find ~/rpmbuild/RPMS -name 'cc-switch-*.rpm' -print
sudo rpm -ivh ~/rpmbuild/RPMS/x86_64/cc-switch-1.0.0-1*.rpm
rpm -qa | grep cc-switch
rpm -ql cc-switch
which cc-switch
cc-switch --version
cc-switch switch rpm-test
cc-switch current

命令与参数解释：

  • rpmbuild -ba 同时构建二进制 RPM 和源码 RPM。
  • rpm -ivh 安装 RPM 包；-i 安装，-v 显示详细过程，-h 显示进度。
  • rpm -ql cc-switch 查看包安装了哪些文件。

图片占位符：RPM 构建与安装验证截图
需要包含 rpmbuild、rpm -ivh、rpm -qa、rpm -ql、cc-switch --version 输出

七、课程设计报告可直接使用的项目描述

本课程设计以 cc switch 命令行工具为适配对象，围绕银河麒麟操作系统的软件迁移与打包规范，分别完成 DEB 与 RPM 两种格式的软件包制作、安装验证、命令运行验证和卸载验证。项目首先对运行环境、CPU 架构和打包工具进行确认，然后整理 cc switch 的可执行文件和说明文档；在 DEB 方向，按照 Debian 包目录规范编写 control 文件并构建软件包；在 RPM 方向，初始化 rpmbuild 构建目录，准备源码压缩包并编写 spec 文件。最终通过 dpkg、rpm、which 和 cc-switch 命令输出证明软件包能够被系统包管理器识别、安装、查询、运行和卸载，达到了国产操作系统软件适配迁移课程设计要求。

注意：本页现在使用真实 Release 二进制文件。若你选择 linux-x64 包，DEB 的 Architecture 建议改为 amd64，RPM 不建议继续使用 BuildArch: noarch，应按实际构建环境生成 x86_64 包；若选择 linux-arm64 包，则 DEB 架构应为 arm64，RPM 架构应对应 aarch64。
