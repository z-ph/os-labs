🧪 实验清单（9.1–9.4）

9.1 调度器（2 个小实验） ▼
9.1.1 基于 Kylin OS 的进程调度与优先级实验
编写多线程高 CPU 负载程序（nice-exp.c），编译运行后观察 top 中的 CPU 占用；绑定 CPU 核心并调整 nice 值，验证 CFS 调度器的时间片分配比例（约 3:1）。
C 语言pthreadnice/renicetasksetCFS
9.1.2 实现 O(1) 调度器算法
使用查表法（OSUnMapTbl[256]）与位运算实现 O(1) 优先级调度，模拟就绪组 OSRdyGrp 和就绪表 OSRdyTbl，快速定位最高优先级任务（无循环遍历）。
C 语言位运算查表法实时调度
9.2 内存管理（3 个小实验，不含 QEMU 安装） ▼
9.2.2 页表快照实验
在 QEMU 虚拟机中运行 pgd-walk 程序，使用 dump-guest-memory 导出物理内存，在 WSL2 Ubuntu 中用 crash 工具手动分析页表，完成虚拟地址 → 物理地址映射。
QEMUWSL2crashMMU页表漫游
9.2.3 Slab 内存分配实验
编写内核模块 slab_test.c，使用 kmem_cache_create 创建 Slab 缓存，kmem_cache_alloc / kmem_cache_free 分配与释放内存，演示链表遍历与安全释放。
内核模块Slab 分配器Linux Kernel
9.2.4 内存回收实验（OOM）
编写用户态程序 oom.c，死循环疯狂申请物理内存；关闭 Swap 后观察系统内存下降，直到触发 OOM Killer，通过 dmesg 查看内核日志确认进程被杀。
内存管理OOM Killerdmesg
9.3 文件系统（2 个小实验） ▼
9.3.1 文件系统底层存储调试与扇区映射实验
使用 debugfs、blkcat、blktrace、blkparse 等工具调试 ext 文件系统，掌握 inode ↔ block ↔ sector ↔ content 的映射关系，监控底层 I/O 流程。
debugfsblktraceblkcatinodeext4
9.3.2 简易文件系统设计实现及高级缓存功能开发
编写一个简易文件系统（如基于 FUSE 或内核模块），实现基本创建/读写功能；进阶可加入 Page Cache 支持，提升文件读写性能。
文件系统FUSEPage Cache内核编程
9.4 银河麒麟高级服务器操作系统管理（3 个小实验） ▼
9.4.1 多用户与权限管理
设置目录拥有者/组（chown）、权限位（chmod 660）、粘滞位（o+t），以及 ACL 扩展权限（setfacl/getfacl），实现 yinhe/kylin/school 用户组的精细化权限控制。
权限管理ACLchmodchown
9.4.2 文件快速定位与管理
熟练使用 find、grep、cat、less、more、head、tail、file、stat 等命令，完成按文件名/类型/大小/内容查找与分页查看。
findgrepstat文件检索
9.4.3 数据检索与处理
使用 grep 正则过滤、wc 统计、sort 排序等工具，对 /etc/passwd 等系统文件进行多条件检索与数据分析，掌握 -A/-B/-C 上下文显示等高级用法。
grep 正则wc文本处理

🎯 课设项目要求参考

课设（9.5 或 9.6）要求按照“完整项目要求”实现。依据《企业导师实践第一课》PDF 中 “03 系统适配迁移” 章节，完整项目应包含以下阶段：

01 系统评估

全面了解应用系统现状（架构、语言、依赖、用户规模），制定迁移策略与计划。

02 适配改造

前端兼容性调整、服务端兼容性改造、数据库/中间件/操作系统特性适配、测试优化。

03 适配优化

联合固件、操作系统、中间件、数据库做深度适配，全方位性能调优。

04 实施验证

在基础平台软硬件上做联合调试，功能、性能、稳定性、安全性严格测试验证。

05 文档与报告

编写测试用例、适配测试报告、打包规范文档，形成完整的项目交付物。
